# 抽象TI 项目说明

## 项目简介
抽象系直播员人格测试，仿照SBTI制作。用户回答16道题，测出自己是哪位抽象系直播员。

## 技术栈
- Vue 3 + Vite
- 纯静态，无后端
- 部署在 Cloudflare Pages

## 文件结构
```
src/
├── data/
│   ├── results.js    <- 20位直播员数据，改这里添加/修改人物
│   └── questions.js  <- 32道题库，改这里添加/修改题目
├── components/
│   ├── AppHeader.vue     <- 顶部标题（静态）
│   ├── QuizIntro.vue     <- 首页开始按钮
│   ├── QuizProgress.vue  <- 进度条
│   ├── QuizQuestion.vue  <- 答题卡
│   └── QuizResult.vue    <- 结果页
├── App.vue       <- 状态管理 + 核心逻辑（抽题、评分、出结果）
├── main.js
└── style.css     <- 全局样式
```

## 数据结构

### results.js — 人物数据
导出具名对象 `R`，每个 key 是人物 ID，value 包含：
- name: 姓名
- alias: 别名
- tags: 标签数组，第一个 tag 会高亮显示
- desc: 人物描述
- quote: 经典语录

### questions.js — 题目数据
导出具名数组 `POOL`，每道题包含：
- q: 题目文字
- opts: 4个选项数组，每个选项包含：
  - t: 选项文字
  - sc: 得分映射对象，key 是人物 ID，value 是分值（强相关用3，弱相关用1）

## 评分机制（App.vue 的 showResult 函数）
归一化评分：最终得分 = 原始得分 / 该人物在本次16题中出现次数

目的：防止高频出现的人物垄断结果，让低频人物也有机会胜出。

## 如何调整某人的出现概率
- 提高：在更多题目选项的 sc 里加入该人物 ID，或把分值从1提高到3
- 降低：减少该人物在 sc 里出现的次数，或把分值从3降到1
- 完全不出现：把该人物从所有 sc 里移除

## 如何添加新直播员
1. 在 results.js 的 R 对象里加一条新人物数据
2. 在 questions.js 的 POOL 里至少给他加4-6个选项的得分，保证有机会被测出来

## 部署
npm run build 生成 dist/ 文件夹，git push 后 Cloudflare Pages 自动部署

## 抽象文化背景

抽象文化起源于斗鱼直播平台6324直播间，由李赣（李老八）于2015年创立抽象工作室，成员包括孙笑川等人。以反逻辑、反严肃、接地气的直播风格著称，形成了独特的网络亚文化圈子。"狗粉丝""嗨粉""KTV（踢出工作室）""节目效果""贵宾""接任务"等都是圈内黑话。

## 常用抽象黑话
- **KTV**：被踢出工作室/直播间
- **接任务**：做某些不方便明说的事
- **节目效果**：直播中的精彩内容/反应
- **贵宾**：花钱刷礼物的付费粉丝
- **狗粉丝/嗨粉**：不刷礼物的普通观众
- **追龙**：赌博
- **嗯格局/嗯冲**：李老八口头禅，表示要有大局观
- **五影会谈**：家庭会议，通常是拷打小孩问成绩
- **破防**：情绪崩溃/被整到
- **整活**：制造节目效果
- **对外软中对内嗯冲**：对外人态度软但内部硬气
- **黄金圣衣**：洪伟强和三文鱼的特殊表演，不建议饭点了解
- **打洞**：石林在越南的特殊赛道
